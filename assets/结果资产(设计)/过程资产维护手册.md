# 过程资产维护手册

本文档用于说明本项目开发过程中如何记录问题、决策和技术专题。它面向未来的自己，也面向上下文压缩后的新对话，用来恢复过程资产的维护节奏。

过程资产不替代正式设计文档。正式设计文档记录当前结论，过程资产记录结论形成过程中的问题、取舍、经验和可复用方法。

## 1. 过程资产定位

过程资产主要解决三个问题：

- 开发中遇到的问题为什么发生、如何修复、如何验证。
- 设计或工程边界为什么这样划分，而不是采用另一种方案。
- 某个工具、框架或机制在当前项目中如何使用，后续项目如何复用。

过程资产不要求记录所有对话，也不要求记录每一次小修。只记录有复盘价值、后续容易再次遇到、能帮助理解项目设计或提升工程判断的内容。

## 2. 三类记录方式

### 2.1 技术实现问题记录

载体：

```text
assets/过程资产/技术实现问题记录.md
```

适合记录具体实现问题，例如：

- 编译失败、启动失败、接口报错。
- JWT 解析、MyBatis-Plus 映射、参数校验等实现坑。
- 具体 Bug 的现象、根因、修复方式和验证方法。

推荐结构：

```md
## N. 问题标题

### 问题类型
### 现象
### 根因
### 修复方式
### 验证方式
### 经验沉淀
```

停止标准：

- 能说清现象、根因、修复、验证即可停止。
- 不展开成教程。
- 不把完整对话粘贴进去。

### 2.2 工程设计决策记录

载体：

```text
assets/过程资产/工程设计决策记录.md
```

适合记录工程原则、设计边界和方案取舍，例如：

- URL、Controller、Service 的职责如何划分。
- 固定角色权限放在拦截器还是业务代码。
- Service 层为什么要重复校验实体业务状态。
- 状态流转、事务边界、模块依赖方向如何确定。

推荐结构：

```md
## N. 决策标题

### 背景
### 冲突点
### 可选方案
### 最终决策
### 适用原则
### 当前项目落点
```

停止标准：

- 能抽象出可复用原则即可停止。
- 不记录所有代码细节。
- 不替代正式设计文档；如果设计结论变化，应同步更新源设计文档。

### 2.3 专题文档

载体：

```text
assets/过程资产/专题文档/
```

适合记录可体系化复用的技术主题，例如：

- Knife4j / OpenAPI 使用方法。
- JWT 鉴权实现笔记。
- MyBatis-Plus 枚举映射笔记。
- Apifox 接口测试流程。

推荐结构：

```md
# 专题名称

## 1. 解决什么问题
## 2. 当前项目如何使用
## 3. 核心配置或代码示例
## 4. 常见问题
## 5. 后续项目复用清单
```

停止标准：

- 能指导后续项目复用即可停止。
- 不追求一次写成完整教程。
- 后续遇到同类问题时追加和精炼。

## 3. 记录判断规则

遇到一个新问题时，先按以下规则分类：

- 如果重点是具体 Bug 或实现坑，写入 `技术实现问题记录.md`。
- 如果重点是边界划分、职责归属、设计取舍，写入 `工程设计决策记录.md`。
- 如果重点是某项技术的系统用法，写入 `专题文档/` 下的专题文档。
- 如果只是临时小错、命名修正、普通代码调整，不记录。

如果一个问题同时具有多种性质，优先记录最有复盘价值的部分。必要时可以在不同文档中互相提及，但不要复制同一段内容。

## 4. 文档维护规则

- 设计结论变化：直接更新对应源设计文档。
- 具体实现问题：追加到 `技术实现问题记录.md`。
- 工程原则和方案取舍：追加到 `工程设计决策记录.md`。
- 工具或框架用法：整理到 `专题文档/`。
- 旧的 `开发排错笔记.md` 保留为历史错题本，不作为后续主要新增入口。

当前阶段允许先追加，V1 完成后再统一精炼。精炼时删除重复内容，保留能解释项目思路、实现质量和踩坑经验的材料。

## 5. 上下文压缩后的接续规则

如果新的对话因上下文压缩而失去前文，应按以下顺序恢复节奏：

1. 读取本文档，确认过程资产分类规则。
2. 查看 `assets/过程资产` 的目录结构。
3. 根据当前问题性质选择记录载体。
4. 只补充有复盘价值的内容，不追求覆盖所有历史对话。
5. 如果用户要求“记录本轮问题”，优先判断是问题卡、工程决策还是专题文档。

默认不要直接复制对话原文。应将问题整理成背景、冲突、结论和经验。

## 6. 当前目录约定

```text
assets/过程资产/
├── 设计与开发问题记录.md
├── 技术实现问题记录.md
├── 工程设计决策记录.md
├── 开发排错笔记.md
└── 专题文档/
```

其中：

- `设计与开发问题记录.md`：偏业务设计、面试素材和项目讲解。
- `技术实现问题记录.md`：偏具体实现问题卡。
- `工程设计决策记录.md`：偏工程边界、职责划分和方案取舍。
- `开发排错笔记.md`：旧式错题本，保留不主动扩展。
- `专题文档/`：沉淀可复用技术专题。
