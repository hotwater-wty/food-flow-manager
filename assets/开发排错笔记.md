# 开发排错笔记

本文档用于记录开发过程中遇到的具体错误、原因定位和修复方式，定位为个人笔记或错题本，不作为面试素材整理。

## 1. Apifox 携带 Token 仍然被拦截器返回 401

### 现象

使用 Apifox 请求受保护接口：

```http
GET /api/user/tables
```

请求中携带了登录接口返回的 JWT，但服务端仍然返回：

```text
401 Unauthorized
```

### 错误写法

在 Header 中传入：

```http
token: <jwt>
```

### 原因

后端拦截器读取的是标准认证请求头：

```java
request.getHeader("Authorization");
```

并且要求格式为：

```http
Authorization: Bearer <jwt>
```

如果传的是 `token` 请求头，后端读取不到 `Authorization`，会认为没有携带令牌并返回 401。

### 正确写法

在 Apifox 的 Header 中手动配置：

```http
Authorization: Bearer <jwt>
```

注意：

- Header 名必须是 `Authorization`。
- 值必须以 `Bearer ` 开头。
- `Bearer` 和 JWT 之间有一个空格。

也可以在 Apifox 的 Auth 面板中选择 `Bearer Token`，只填写 JWT 本体，由 Apifox 自动生成请求头。

### 经验

JWT 本身生成成功不代表请求一定能通过鉴权。排查 401 时要先检查：

- 请求路径是否被拦截器保护。
- Header 名是否和后端读取的一致。
- Token 格式是否包含 `Bearer ` 前缀。
- Token 是否过期。
- Token 中是否包含后端需要的用户标识字段。
